Lambda表达式使用场合

使用条件：

  1. 实现的对象是函数式接口的抽象方法，比如Runnable的run方法
  2. 函数式接口的抽象方法的函数描述和Lambda表达式函数描述符一致

接口可以有默认方法，而且可以有多个，这不影响接口是否为函数式接口，但是抽象方法只能有一个。

函数描述符

函数描述符也可以理解为方法的签名。比如上述的Runnable抽象方法不接受参数，并且返回void，所以其函数描述符为 () -> void 。而 () -> System.out.println("hello") Lambda表达式也是不接受参数，并且返回void，所以其函数描述符也是 () -> void 。所以代码 Runnable r = () -> System.out.println("hello"); 是合法的。

虽然好像和Lambda表达式的3个结构不同，这种写法也有它的名称：方法的引用

方法引用可以被看作仅仅调用特定方法的Lambda的一种快捷写法。它的基本思想是，如果一个Lambda代表的只是“直接调用这个方法”，那最好还是用名称来调用它，而不是去描述如何调用它，这样代码可读性更好。

方法引用格式为：目标引用::方法名称

常见的方法引用：

Lambda表达式 等效方法引用
(String s) -> System.out.println(s) System.out::println
(str, i) -> str.substring(i) String::substring
() -> Thread.currentThread().dumpStack() Thread.currentThread()::dumpStack
